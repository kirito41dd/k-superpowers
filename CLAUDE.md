# K Superpowers

## 项目记忆机制

本项目使用 `.ai/memory/` 管理长期记忆，采用**渐进式披露**：不要预读所有记忆文件，按需加载。

### 启动流程

新会话或任务开始时：
1. **只读** `.ai/memory/index.md`（含触发条件和标签索引）
2. 根据当前任务关键词，判断需要读取哪些具体记忆文件
3. 只读相关文件，不要一次性读取整个 `.ai/memory/` 目录
4. 读取前先看文件头部 `<!-- SUMMARY -->` 块判断相关性

### 按需加载示例

- 用户问“为什么这么做” / 做技术选型 → 读 `decisions.md`
- 用户提新需求 / 讨论功能范围 → 读 `requirements.md`
- 调试 bug / 修改敏感模块前 → 读 `gotchas.md`
- 写新代码 / 用户纠正风格 → 读 `conventions.md`
- 遇到陌生术语 → 读 `glossary.md`

### 写入流程

识别到值得记忆的内容时（决策/需求/坑/约定/术语），**先询问用户**，确认后：

1. 查 `index.md` 找到对应文件
2. 追加到文件顶部（倒序时间线）
3. 同步更新 `index.md` 的“最近热点”“标签索引”和“快速索引”
4. 同步更新记忆文件的 `SUMMARY` 块
5. 文件超过 500 行时归档最旧内容到 `archive/<year>/<filename>`

触发话术：
> 这个 [决策/需求/坑] 值得记入项目记忆吗？我会追加到 `.ai/memory/xxx.md`。

### 维护边界

- 不删除历史记忆，过时标记 `[DEPRECATED]`
- 避免跨文件重复，用完整锚点引用
- 写入前先搜索现有内容，重复则更新而非新增
- `glossary.md` 写入后需按字母顺序重排序
